# Researcher - 论文调研

> 一句话：输入主题，输出报告。

## 命令

| 命令 | 功能 | 适合场景 |
|------|------|----------|
| `/researcher 主题` | 调研主题 → 输出报告 | 多数情况用这个 |
| `/researcher-deep 主题` | 深度调研（搜索+阅读+详细报告） | 写论文、做综述 |
| `/researcher-teach 概念` | 讲解概念 | 不懂的地方问这个 |

## 默认路径（/researcher 主题）

**目标**：用户输入主题，直接给结果，不废话。

```
用户：/researcher RAG最新进展
→ 5-8轮搜索
→ 整理为markdown报告
→ 输出到 /mnt/agents/output/report.md
```

**期间只打断一次**：确认搜索范围（可选）
- "时间范围偏好？（默认最近1年）"
- 用户可直接回车跳过，用默认值

**报告内容**：
- 执行摘要（2-3段）
- 关键论文列表（标题/年份/核心贡献）
- 主要发现
- 来源

**不做的**（除非用户要求）：
- 不逐篇深度阅读论文
- 不构建知识图谱
- 不教学讲解
- 不多模态文件处理

## 深度路径（/researcher-deep 主题）

**采访模式**（3个问题，每个可直接跳过）：

| # | 问题 | 默认 | 影响 |
|---|------|------|------|
| 1 | "时间范围？（最近1年/3年/不限）" | 不限 | 搜索范围 |
| 2 | "需要哪些高级功能？"见下表 | 仅报告 | 启用子功能 |
| 3 | "详细程度？（简要/标准/详细）" | 标准 | 报告深度 |

**高级功能选择**（问题2，多选）：

| 功能 | 说明 | 增加时间 |
|------|------|----------|
| 📄 逐篇深度阅读 | 每篇论文结构化提取（方法/实验/创新点） | +10-20min |
| 🕸️ 引文网络分析 | 从一篇种子论文沿引用网络发现关联论文（类似Connected Papers） | +5min |
| 📊 对比表格 | Elicit-style合成数据表 | +2min |
| 🧠 知识图谱 | 概念关联网络（Zettelkasten） | +3min |
| 🖼️ 多模态处理 | 处理用户上传的PDF/图片 | 视文件数 |
| 👨‍🏫 概念教学 | 对关键概念进行讲解 | +5min/概念 |

引文网络分析详情见 `references/citation-network.md`。

**流程**：
```
用户：/researcher-deep RAG
→ Q1: 时间范围？（用户回车=不限）
→ Q2: 勾选高级功能（用户只勾"逐篇阅读+对比表格"）
→ Q3: 详细程度？（用户回车=标准）
→ 执行：搜索 → 阅读 → 表格 → 报告
→ 输出 report.md
```

## 教学路径（/researcher-teach 概念）

直接讲解，不调研。

```
用户：/researcher-teach 什么是注意力机制
→ 问：背景水平？（初级/中级/高级/回车=自动判断）
→ 苏格拉底式讲解
→ 生成概念卡片 → 可选保存到知识图谱
```

## 文件结构（深度模式才创建）

```
research/{topic-slug}/           # 深度模式才创建此目录
  report.md                      # 主报告（始终生成）
  papers/                        # 逐篇阅读结果（可选）
  knowledge-graph.md             # 知识图谱（可选）
  raw-extracts/                  # 多模态提取（可选）
  image-descriptions/            # 图片分析（可选）
```

**默认模式**（/researcher）只生成 `report.md` 到 `/mnt/agents/output/`，不创建研究目录。

## 核心原则

1. **默认极简**：`/researcher` 直接给结果，不问一堆问题
2. **深度可选**：`/researcher-deep` 才展开高级功能菜单
3. **按需付费**：每个子功能独立开关，用户只选需要的
4. **可跳过**：所有问题都有默认值，回车即跳过
5. **一次打断**：默认路径最多打断1次，深度路径最多3次
