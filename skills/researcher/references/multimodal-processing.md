# 多模态输入处理（Phase 0.5 完整规范）

本文档是 Phase 0.5（多模态输入处理）的权威参考。当用户上传文件或提供 URL 时激活。

## 文件类型路由

| 输入 | 扩展名 | 处理工具 | 输出位置 |
|------|--------|----------|----------|
| PDF 论文 | .pdf | Read | `raw-extracts/pdf-*.md` |
| Word 文档 | .docx/.doc | Read | `raw-extracts/word-*.md` |
| 文本/Markdown | .md/.txt/.json/.yaml | Read | `raw-extracts/text-*.md` |
| 论文截图/插图 | .png/.jpg/.jpeg/.gif/.webp/.svg | 视觉分析 | `image-descriptions/img-*.md` |
| 网页链接 | URL | WebFetch | `raw-extracts/web-*.md` |
| 图片 URL | http*.jpg/png | WebFetch + 视觉分析 | `image-descriptions/img-*.md` |

**注意**：如果有专门的 skill（如 `pdf`、`docx-parse-tool`），优先用 skill 而非裸读，避免重复造轮子。

## 执行流程

1. **检测输入**：列出用户提供的所有文件和链接
2. **分类路由**：按上表路由到对应处理工具
3. **逐个处理**：
   - 文本文件 → Read → 提取为 markdown → `raw-extracts/{filename}-extract.md`
   - 图片/截图 → 视觉分析 → 结构化描述 → `image-descriptions/{desc}.md`
   - 网页 → WebFetch → 关键内容 → `raw-extracts/web-{slug}.md`
4. **统一索引**：所有提取内容注册到 `sources-index.md`
5. **确认完整性**：用户确认提取结果，遗漏的补充处理

## 视觉分析能力（论文插图）

我能直接"看"图片并分析：

| 图片类型 | 分析内容 |
|----------|----------|
| 模型架构图 | 模块名称、数据流向、层间连接 |
| 实验结果图 | 横纵轴含义、数据趋势、关键数值点 |
| 注意力热力图 | 注意力分布模式解读 |
| 损失曲线图 | 收敛性、过拟合信号 |
| 对比表格图 | OCR 提取表格数据 |
| 数学公式 | OCR → LaTeX 格式 + 含义解释 |
| 流程图/算法图 | 步骤识别 → 伪代码/文字描述 |

## 图片分析深度模板

```markdown
# 图片分析: {图片标题或编号}

## 来源
- **论文**: {论文标题}
- **章节**: {Figure X, Page Y}
- **原始文件**: {原始文件名}

## 图片描述
{视觉分析：图中展示的内容、各模块名称、数据流向}

## 关键信息提取
- {数据点1}: {数值/结论}
- {数据点2}: {数值/结论}
...

## 与论文文字的对应
{图中信息与论文正文描述的关联}

## 个人解读
{对图表含义的分析、与同类图表的对比}
```

## 批量处理策略

用户一次性提供多个文件时：

1. **分类**：按文件类型分组
2. **优先级**：
   - P0: 核心论文 PDF（最优先处理）
   - P1: 补充论文/综述 PDF
   - P2: Word 文档、网页
   - P3: 图片/截图（需逐张分析，耗时较长）
3. **并行**：文本类文件可批量处理，图片类需逐个分析
4. **索引**：每处理完一个文件，立即更新 `sources-index.md`

## 常见问题处理

| 问题 | 原因 | 处理 |
|------|------|------|
| PDF 提取不完整 | 扫描版 PDF（图片形式） | 提示用户，建议截图关键页面 |
| 图片分辨率过低 | 压缩或缩小 | 标记在备注中，建议获取高清版 |
| 网页抓取失败 | 反爬虫/需登录 | 尝试替代源；标记为受限 |
| 公式识别错误 | 复杂排版 | LaTeX 语法检查，与正文交叉验证 |
