# 报告生成规范（/researcher-report 的权威参考）

本文档是 Phase 4（综合与报告）的完整规范，包含交叉验证规则、来源质量排序、偏见检查、报告骨架和组织方式选择。

## 交叉验证

1. 读取所有 `notes/*.md` 和 `papers/*.json`
2. 对同一事实，检查多个来源是否一致
3. 标记：✅ 多方确认 / ⚠️ 单一来源 / ❌ 矛盾

矛盾处理优先级（来源质量排序）：

| 优先级 | 来源类型 | 说明 |
|--------|----------|------|
| 1 | 同行评审论文 | 最可信 |
| 2 | 政府/官方数据 | 统计可靠 |
| 3 | 行业报告 | 有商业偏见可能 |
| 4 | 博客/技术文章 | 可作为补充 |
| 5 | 单一论文声称 | 需标注"待验证" |

## 偏见检查

报告生成前检查：
- **搜索偏见**：搜索结果是否偏向某一观点？
- **时间偏见**：过度依赖最新论文而忽略经典？
- **来源偏见**：是否主要来自同一机构/团队？
- **确认偏见**：是否只收集了支持预设假设的证据？

若发现偏见，在"局限性与注意事项"中明确说明。

## 共识 vs 不确定性

- **共识区**：多论文一致确认的结论 → 报告核心发现
- **争议区**：论文间存在分歧 → 报告"分析"部分对比呈现
- **不确定区**：证据不足或矛盾 → 报告"局限性与注意事项"

## 报告骨架

```markdown
# {topic} - 文献综述报告

## 执行摘要
[2-3 段关键发现概述，含范围、方法、核心结论]

## 论文概览
| 序号 | 论文 | 年份 | 会议 | TL;DR | 质量 |
[全部论文的缩略表]

## 关键发现
### [发现1标题]
[讨论 + 内联引用]

### [发现2标题]
...

## 分析
### 跨论文模式
[方法论趋势、技术路线演进]

### 矛盾与共识
[不同论文间的分歧和一致点]

## 局限性与注意事项
[搜索范围限制、语言限制、时间限制、来源偏见]

## 结论

## 来源索引
[编号来源列表]
```

## 组织方式选择

用 AskUserQuestion 确认，默认"按时间线"：

| 方式 | 适用场景 |
|------|----------|
| 按时间线 | 技术演进综述、历史回顾 |
| 按主题分类 | 多维对比综述、方法分类 |
| 按重要性 | 核心工作优先、快速了解全貌 |

## 详细程度选择

| 级别 | 包含内容 |
|------|----------|
| 简要 | 仅核心方法、创新点、主要结果 |
| 标准（默认）| 除研究动机和实验数据集外全部 |
| 详细 | 全部字段，含研究动机和消融实验 |

## Output Validation（输出验证）

报告生成后执行结构化质量检查，防止错误传播。

| 检查项 | 标准 | 未通过处理 |
|--------|------|-----------|
| TL;DR 完整性 | 100% 论文覆盖 | 回到 Phase 3 补充缺失项 |
| 数字来源 | 所有关键数字标注 `[论文, 年份]` | 追溯来源或标注 `[待验证]` |
| 矛盾公平呈现 | 无遗漏的对立观点 | 回到 Phase 2 补充搜索 |
| 偏见说明 | 局限性章节包含偏见评估 | 补充偏见分析段落 |
| 来源索引对应 | 所有引用有 sources-index 条目 | 补充缺失条目 |
| 摘要独立性 | 仅读摘要能理解核心结论 | 重写执行摘要 |

验证结果写入 `checkpoint.md` 的 C4 检查点。

## 质检清单

报告发布前确认：
- [ ] 每篇论文都有 TL;DR
- [ ] 所有关键数字有来源标注
- [ ] 矛盾观点被公平呈现
- [ ] 偏见在局限性中说明
- [ ] 来源索引与正文引用对应
- [ ] 执行摘要能独立传达核心结论
